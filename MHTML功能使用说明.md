# MHTML文件功能使用说明

## 功能说明

为了应对某些网站的反爬虫保护，本工具现在支持从MHTML文件中读取数据。

### 什么是MHTML？

**MHTML** (MIME HTML) 是一种网页保存格式，包含：
- 完整的HTML内容
- 所有嵌入的资源（图片、CSS、JavaScript等）
- 可以在浏览器中完美还原原始网页

所有现代浏览器都支持将网页保存为MHTML格式。

---

## 使用步骤

### 方法1：Chrome浏览器保存MHTML

1. **打开网页**
   - 在Chrome浏览器中打开目标网页（如看准网、脉脉等）
   - 等待页面完全加载

2. **保存为MHTML**
   - 按 `Ctrl + S` 或右键点击页面
   - 选择"另存为"
   - 在保存类型中选择"网页，单个文件（*.mhtml)"
   - 保存文件

3. **在工具中使用**
   - 打开"公司文化职位解析工具"
   - 在"数据来源"中选择 **"MHTML文件"**
   - 点击"选择文件"按钮
   - 可以一次选择多个MHTML文件
   - 继续选择简历文件和输入JD
   - 点击"开始分析"

### 方法2：Edge浏览器保存MHTML

1. 打开目标网页
2. 按 `Ctrl + S`
3. 保存类型选择"网页，单个文件"
4. 保存为 `.mhtml` 文件

### 方法3：Firefox浏览器保存MHTML

Firefox需要安装扩展才能保存MHTML：
1. 安装 "UnMHT" 扩展
2. 右键点击页面 → "使用UnMHT保存"
3. 选择保存位置

## 优势

### ✅ 避开反爬虫保护
- 不受网站反爬虫限制
- 不需要担心IP被封
- 完全模拟真实用户浏览

### ✅ 数据完整
- 保存网页的完整状态
- 包括动态加载的内容
- 不会遗漏任何信息

### ✅ 灵活便捷
- 可以提前准备好数据
- 支持批量处理多个文件
- 便于数据归档和重复使用

## 对比：网页抓取 vs MHTML文件

| 特性 | 网页抓取 | MHTML文件 |
|------|---------|-----------|
| **速度** | 较慢（需要等待加载） | 快速（直接读取） |
| **成功率** | 受反爬虫影响 | 100%（用户保存） |
| **数据完整性** | 可能遗漏动态内容 | 完整（浏览器保存） |
| **适用场景** | 简单网站 | 严格保护的网站 |
| **批量处理** | 需要多次访问 | 可批量读取文件 |
| **网络要求** | 需要稳定网络 | 离线可用 |

## 使用建议

### 推荐使用MHTML文件的场景

1. **严格保护的网站**
   - 看准网（被Tencent Cloud EdgeOne保护）
   - 脉脉（需要登录）
   - 其他有反爬虫机制的网站

2. **需要多次分析**
   - 保存一次，多次使用
   - 避免重复访问网站

3. **批量分析**
   - 一次性保存多个公司的网页
   - 批量导入工具分析

4. **离线工作**
   - 无需网络连接
   - 在任何地方都能分析

### 推荐使用网页抓取的场景

1. **简单的公开网站**
   - 没有反爬虫保护
   - 内容较少，加载快

2. **实时性要求高**
   - 需要获取最新数据
   - 不想手动保存

## 常见问题

### Q1: MHTML文件保存在哪里？

**A**: 任意位置都可以。工具会记住上次选择的目录。

建议：
- 创建专门的文件夹（如 `公司评价数据/`）
- 按公司名称分类保存
- 便于批量选择和管理

### Q2: 一次可以选择多少个MHTML文件？

**A**: 没有限制。可以一次选择几十个甚至上百个文件。

工具会：
- 自动批量读取所有文件
- 合并所有内容进行分析
- 在日志中显示每个文件的处理结果

### Q3: MHTML文件可以和网址混用吗？

**A**: 不可以。每次只能选择一种数据源：
- 要么使用"网页抓取"（输入网址）
- 要么使用"MHTML文件"（选择文件）

### Q4: 如何批量保存网页为MHTML？

**Chrome批量保存方法：**
1. 安装Chrome扩展 "SingleFile" 或 "Save MHTML"
2. 打开多个网页，逐个保存
3. 或者使用浏览器插件批量保存

**手动批量保存：**
1. 依次打开每个网页
2. Ctrl+S 保存为MHTML
3. 放到同一文件夹
4. 在工具中一次性全选

### Q5: MHTML文件会过期吗？

**A**: 不会。MHTML文件是静态文件，可以永久保存和重复使用。

## 技术细节

### MHTML格式结构

```
Content-Type: multipart/related;
    boundary="----=_boundary"

------=_boundary
Content-Type: text/html

[HTML内容]

------=_boundary
Content-Type: image/jpeg
Content-Transfer-Encoding: base64

[图片数据]

------=_boundary
Content-Type: text/css

[CSS样式]

------=_boundary--
```

### 工具如何处理MHTML

1. **读取文件**
   - 自动检测编码（UTF-8/GBK）
   - 解析MHTML格式

2. **提取内容**
   - 提取HTML部分
   - 移除脚本和样式
   - 提取纯文本内容

3. **解析信息**
   - 提取页面标题
   - 提取原始URL
   - 提取正文内容

4. **数据清理**
   - 解码HTML实体
   - 清理多余空白
   - 合并段落

## 故障排查

### 问题1: 读取MHTML文件失败

**可能原因**：
- 文件损坏
- 编码不支持

**解决方法**：
- 重新保存MHTML文件
- 尝试用不同的浏览器保存

### 问题2: 内容不完整

**可能原因**：
- 保存时页面未完全加载

**解决方法**：
- 等待页面完全加载后再保存
- 手动滚动到页面底部
- 确保所有内容都已显示

### 问题3: 包含大量无用文本

**可能原因**：
- 页面包含大量脚本/样式代码

**解决方法**：
- 工具会自动清理大部分无用内容
- 如果仍有问题，可以手动编辑MHTML文件

## 示例操作流程

### 完整流程示例

假设要分析腾讯、阿里、字节三家公司：

1. **准备MHTML文件**
   ```
   公司评价数据/
   ├── 腾讯-看准网.mhtml
   ├── 阿里-看准网.mhtml
   └── 字节-看准网.mhtml
   ```

2. **打开工具**
   - 启动"公司文化职位解析工具.exe"

3. **选择数据源**
   - 数据来源：选择 **"MHTML文件"**

4. **选择MHTML文件**
   - 点击"选择文件（可多选）"
   - 框选三个.mhtml文件
   - 点击"打开"
   - 显示"已选择 3 个文件"

5. **选择简历**
   - 点击"浏览..."选择简历文件

6. **输入JD**
   - 粘贴职位描述

7. **开始分析**
   - 点击"开始分析"
   - 等待2-3分钟
   - 查看生成的HTML报告
