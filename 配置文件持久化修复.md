# 配置文件持久化修复报告

## 修复时间
2026-01-12 14:20

## 问题描述

用户报告："重新构建exe之后，没有读取之前用户的界面信息"

### 根本原因

在 `main.py` 第30行，配置文件使用相对路径：

```python
self.config_file = Path("config.json")
```

**问题**：
- 当从EXE运行时，工作目录可能不是EXE所在目录
- 用户双击EXE运行时，当前工作目录可能变化
- 导致配置文件找不到或保存到错误的位置

---

## 修复方案

### 1. 使用绝对路径

修改 `main.py` 的 `__init__` 方法，根据运行环境自动确定配置文件路径：

```python
# 配置文件路径 - 使用绝对路径，确保EXE运行时能找到
if getattr(sys, 'frozen', False):
    # 如果是EXE运行，使用EXE所在目录
    application_path = Path(sys.executable).parent
else:
    # 如果是Python脚本运行，使用脚本所在目录
    application_path = Path(__file__).parent

self.config_file = application_path / "config.json"
```

**优势**：
- ✅ EXE运行：配置文件保存在EXE同目录
- ✅ Python运行：配置文件保存在脚本同目录
- ✅ 无论从哪里启动，都能正确找到配置文件

## 技术细节

### `sys.frozen` 检测

```python
if getattr(sys, 'frozen', False):
    # EXE模式
    application_path = Path(sys.executable).parent
else:
    # Python脚本模式
    application_path = Path(__file__).parent
```

**原理**：
- PyInstaller打包的EXE会设置 `sys.frozen = True`
- `sys.executable` 指向EXE文件的完整路径
- Python脚本运行时，`__file__` 指向脚本文件的完整路径

### 路径拼接

使用 `pathlib.Path` 的 `/` 操作符：

```python
config_file = application_path / "config.json"
```

**优势**：
- 自动处理不同操作系统的路径分隔符
- 避免硬编码分隔符（`\\` 或 `/`）
- 更安全，不会出现路径拼接错误

## 常见问题

### Q1: 配置文件找不到？

**原因**：从不同目录启动程序

**解决**：使用绝对路径（已修复）

**原因**：文件编码不是UTF-8

**解决**：
- 程序强制使用UTF-8读写
- 手动编辑时确保保存为UTF-8编码
