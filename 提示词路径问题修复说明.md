# 提示词文件路径问题修复说明

## 问题描述

用户测试批量工具时遇到错误：
```
处理失败: 未找到提示词文件 prompts2/综合评估与排名.txt
```

尽管文件存在于 `C:\D\CAIE_tool\MyAIProduct\position\prompts2\综合评估与排名.txt`，程序运行时却找不到。

## 根本原因

**路径解析问题**：原代码在 `batch_processor.py` 第187行使用相对路径：
```python
prompt_file = Path("prompts2/综合评估与排名.txt")
```

这个相对路径在以下场景会有问题：
1. **Python开发模式**：依赖当前工作目录（CWD），如果从其他目录运行会失败
2. **EXE打包模式**：PyInstaller将文件打包到临时目录 `_MEIPASS`，相对路径无法解析

## 修复方案

在 `batch_processor.py` 的 `generate_ranking_report()` 方法中实现了**智能路径查找**：

### 1. 多路径尝试机制

尝试多个可能的路径位置：
```python
possible_paths = [
    Path("prompts2/综合评估与排名.txt"),  # 当前目录
    script_dir / "prompts2" / "综合评估与排名.txt",  # 脚本目录
    Path(__file__).parent / "prompts2" / "综合评估与排名.txt",  # __file__所在目录
]

# 如果是EXE，尝试特殊路径
if getattr(sys, 'frozen', False):
    exe_dir = Path(sys.executable).parent
    possible_paths.append(exe_dir / "prompts2" / "综合评估与排名.txt")
    if hasattr(sys, '_MEIPASS'):
        meipass_dir = Path(sys._MEIPASS)
        possible_paths.append(meipass_dir / "prompts2" / "综合评估与排名.txt")
```

### 2. 详细错误日志

如果所有路径都失败，生成详细的诊断信息：
```python
error_details = f"未找到提示词文件 prompts2/综合评估与排名.txt\n\n"
error_details += f"当前工作目录: {current_dir}\n"
error_details += f"脚本所在目录: {script_dir}\n"
error_details += f"尝试过的路径:\n"
for p in possible_paths:
    error_details += f"  - {p} (存在: {p.exists()})\n"
```

错误信息会保存到 `logs2/path_error_YYYYMMDD_HHMMSS.txt` 文件中。

### 3. 关键路径说明

**sys._MEIPASS**：
- PyInstaller打包后，所有 `--add-data` 数据文件被解压到临时目录
- 这个路径在运行时通过 `sys._MEIPASS` 获取
- 打包时使用的命令：
  ```bash
  --add-data "prompts2;prompts2"
  ```
  这会创建 `_MEIPASS/prompts2/综合评估与排名.txt`
