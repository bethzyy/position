# 网页抓取改进 - 模拟人工下拉滚动

## 改进时间
2026-01-12

## 改进目标

让网页抓取更像人工操作，通过模拟真实用户的滚动行为，避免被网站检测为爬虫。

---

## 改进前 vs 改进后

### 改进前（简单滚动）

```python
# 直接滚动到页面底部
self.driver.execute_script("window.scrollTo(0, document.body.scrollHeight);")

# 固定等待2秒
time.sleep(2)

# 检查页面高度是否变化
```

**问题:**
- ❌ 滚动方式机械，容易被识别为爬虫
- ❌ 固定等待时间，不自然
- ❌ 一次性滚到底部，可能遗漏中间加载的内容
- ❌ 停止条件简单（只检查高度变化）

---

### 改进后（模拟人工滚动）

```python
# 1. 分多次小幅度滚动（而不是直接滚到底部）
current_scroll_position = self.driver.execute_script("return window.pageYOffset || document.documentElement.scrollTop;")
viewport_height = self.driver.execute_script("return window.innerHeight")

# 每次滚动视口高度的0.5-1倍（随机变化）
scroll_distance = int(viewport_height * (0.5 + (scroll_attempts % 3) * 0.2))
new_position = min(current_scroll_position + scroll_distance, page_height)

# 2. 使用平滑滚动（模拟人工）
self.driver.execute_script(f"window.scrollTo({{top: {new_position}, behavior: 'smooth'}});")

# 3. 随机等待1-3秒（模拟人工阅读速度）
wait_time = random.uniform(1.0, 3.0)
time.sleep(wait_time)

# 4. 智能判断停止条件
# - 检查是否到达页面底部
# - 检查连续无变化的次数
# - 最多滚动100次
```

**优势:**
- ✅ 更像真人操作，难以被识别
- ✅ 平滑滚动，视觉自然
- ✅ 随机等待时间，行为不可预测
- ✅ 智能停止，节省时间

---

## 详细改进说明

### 1. 分段式滚动

**原理**: 真人不会一次性滚到底部，而是分段阅读。

```python
# 计算每次滚动距离
scroll_distance = int(viewport_height * (0.5 + (scroll_attempts % 3) * 0.2))
```

- 第1次: 0.5倍视口高度
- 第2次: 0.7倍视口高度
- 第3次: 0.9倍视口高度
- 第4次: 0.5倍（循环）
- ...

**效果**: 滚动距离随机变化，模拟人工的不规则性。

---

### 2. 平滑滚动

```python
self.driver.execute_script(f"""
    window.scrollTo({{
        top: {new_position},
        behavior: 'smooth'
    }});
""")
```

**对比**:
- `behavior: 'smooth'` - 平滑滚动，有动画效果
- 默认（无behavior） - 瞬间跳转，不自然

**效果**: 滚动过程有动画，更接近真实用户行为。

---

### 3. 随机等待时间

```python
wait_time = random.uniform(1.0, 3.0)
time.sleep(wait_time)
```

**范围**: 1-3秒之间的随机浮点数

**目的**:
- 模拟人工阅读速度
- 每次等待时间不同，避免规律性
- 给网站充足时间加载新内容

---

### 4. 智能停止条件

#### 条件1: 到达页面底部且无新内容

```python
at_bottom = (current_position + viewport_height) >= page_height - 50
if at_bottom and no_change_count >= max_no_change:
    break
```

**逻辑**:
- 检查当前位置是否接近页面底部
- 连续3次滚动后页面高度都没有增长
- 说明真的到底了，停止滚动

#### 条件2: 连续无新内容加载

```python
if no_change_count >= max_no_change * 2:  # 6次无变化
    break
```

**逻辑**:
- 即使没到底部，如果连续6次滚动都没有新内容
- 说明页面已经加载完毕
- 提前停止，节省时间

#### 条件3: 达到最大滚动次数

```python
max_scroll_attempts = 100  # 从50增加到100
```

**逻辑**:
- 防止无限循环
- 100次足够滚动很长的页面

---

### 5. 详细的日志记录

```python
self.logger.debug(f"滚动: {current_scroll_position} -> {new_position} (距离: {scroll_distance}px)")
self.logger.debug(f"页面高度增长: {new_height}px")
self.logger.debug(f"页面高度未变化 ({no_change_count}/{max_no_change})")
self.logger.info(f"已到达页面底部且无新内容加载，停止滚动（共{scroll_attempts}次）")
```

**作用**:
- DEBUG级别: 记录每次滚动的详细信息
- INFO级别: 记录关键节点
- WARNING级别: 记录异常情况（达到上限）

## 实际效果

### 场景1: 短页面（内容少）

**改进前**:
```
滚动50次 × 2秒 = 100秒
```

**改进后**:
```
滚动约5-10次 × 平均2秒 = 10-20秒
提前检测到无新内容，快速停止
```

**提升**: 节省80-90%时间

---

### 场景2: 长页面（内容多，有无限加载）

**改进前**:
```
滚动50次 = 100秒
可能无法加载全部内容
```

**改进后**:
```
滚动30-50次 × 平均2秒 = 60-100秒
加载更全面，更接近人工浏览
```

**提升**: 加载更完整，更难被检测

---

### 场景3: 有反爬机制的网站

**改进前**:
```
❌ 机械滚动，容易被识别
❌ 固定节奏，规律性强
❌ 可能被阻止或返回空内容
```

**改进后**:
```
✅ 平滑滚动，模拟真人
✅ 随机等待，难以预测
✅ 分段滚动，行为自然
✅ 成功率更高
```

## 技术细节

### 1. 获取滚动位置

```javascript
// 兼容不同浏览器
window.pageYOffset || document.documentElement.scrollTop
```

### 2. 获取页面尺寸

```javascript
// 页面总高度
document.documentElement.scrollHeight

// 视口高度
window.innerHeight
```

### 3. 判断是否到底部

```python
# 允许50px误差（某些页面底部可能有空白）
at_bottom = (current_position + viewport_height) >= page_height - 50
```
