=======================================
批量公司文化职位解析工具
快速使用指南
=======================================

## 一、首次使用

### 1. 安装依赖
pip install anthropic PyPDF2 python-docx openpyxl pyinstaller

### 2. 设置API密钥
set ZHIPU_API_KEY=你的API密钥

或设置为系统环境变量(推荐)

## 二、准备数据

### 公司资料目录结构
companies/
├── 腾讯/
│   ├── reviews.html (员工评价网页)
│   └── jd.txt (职位描述)
├── 阿里/
│   ├── reviews.html
│   └── jd.txt
└── 字节/
    ├── reviews.html
    └── jd.txt

注意事项:
- 每个公司一个子目录
- 至少包含一个网页文件(.html/.mhtml)
- 必须包含jd.txt文件

## 三、运行程序

### 方式1: 双击启动(推荐)
批量工具启动.bat

### 方式2: 命令行
python batch_tool_main.py

## 四、操作步骤

1. 选择"公司资料目录" (包含多个公司子目录的根目录)
2. 选择"简历文件" (PDF/Word/TXT)
3. 选择"报告输出目录"
4. 点击"开始批量处理"
5. 等待处理完成
6. 查看生成的报告

## 五、输出结果

在输出目录会生成:
- 腾讯.html (腾讯详细分析报告)
- 阿里.html (阿里详细分析报告)
- 字节.html (字节详细分析报告)
- 公司推荐排名.html (综合排名报告)

## 六、测试

### 快速测试(2分钟)
cd tests2
python quick_test.py

### 完整测试(5-10分钟)
cd tests2
python test_batch_tool.py

## 七、生成EXE

build_batch_tool_exe.bat

生成的EXE在 dist/批量职位解析工具.exe

## 八、常见问题

Q: 提示"未找到ZHIPU_API_KEY"
A: 设置环境变量: set ZHIPU_API_KEY=你的密钥

Q: 找不到公司报告
A: 检查公司目录结构,确保每个公司子目录都有jd.txt和网页文件

Q: AI调用失败
A: 检查网络连接和API密钥是否有效,查看logs2/目录中的日志

Q: 程序无响应
A: AI调用需要一定时间,请耐心等待,查看进度条和日志

## 九、重要提示

1. 原有的"公司文化职位解析工具"不受影响
2. 所有新功能都是独立模块
3. API密钥安全存储在环境变量中
4. 所有AI调用都有完整日志记录在logs2/
