## 主要改进

### 1. 移除大标题 ✅

**修改前:** 说明文字显示在输入框右侧
**修改后:** 说明文字显示在输入框下方

**具体调整:**
- 公司资料目录: 说明文字移到下方

### 3. 新增缓存功能 ✅

#### 缓存选项
添加了"缓存设置"选项,用户可以选择:
- ✅ 使用缓存(默认): 如果某公司的报告已存在,跳过重新生成
- ❌ 不使用缓存: 强制重新生成所有报告

#### 智能缓存检查
当用户选择"不使用缓存"时:
1. 自动检查输出目录中已存在的公司报告
2. 如果发现已存在的报告,弹窗询问用户
3. 显示已存在的公司列表(最多显示5个)
4. 询问是否删除并重新生成

**用户选择:**
- 点击"是": 删除旧报告并重新生成(实际不删除,直接覆盖)
- 点击"否": 使用缓存,跳过这些公司

#### 进度显示
- 使用缓存时显示: "使用缓存: [公司名]"
- 正在处理时显示: "正在处理: [公司名]"
- 完成时显示: "完成: [公司名]"

## 使用场景

### 场景1: 首次批量处理
1. 选择公司资料目录(包含多个公司)
2. 选择简历文件
3. 选择输出目录
4. 勾选"使用缓存"
5. 点击"开始批量处理"
6. 所有公司报告和综合排名报告生成完成

### 场景2: 添加新公司后继续处理
1. 在公司资料目录中添加新公司
2. 保持其他设置不变
3. 确保勾选"使用缓存"
4. 点击"开始批量处理"
5. 已存在的公司使用缓存,只处理新公司
6. 重新生成综合排名报告

### 场景3: 强制重新生成所有报告
1. 取消勾选"使用缓存"
2. 点击"开始批量处理"
3. 如果有旧报告,弹窗询问是否重新生成
4. 点击"是"重新生成所有报告
5. 或点击"否"使用缓存

### 场景4: 重新生成特定公司报告
1. 手动删除输出目录中特定公司的报告文件
2. 勾选"使用缓存"
3. 点击"开始批量处理"
4. 只重新生成被删除的公司报告
5. 其他公司使用缓存

## 技术实现

### GUI修改
- 移除标题Label
- 创建独立的path_row框架
- 说明文字使用anchor=tk.W左对齐
- 添加wraplength参数防止文字截断

### 缓存逻辑
1. GUI层检查缓存状态
2. 检测已存在的报告文件
3. 弹窗询问用户
4. 传递use_cache参数给processor
5. Processor在处理时检查报告文件是否存在

### 关键代码

**GUI缓存检查:**
```python
if not use_cache:
    # 检查已存在的报告
    for company_dir in companies_root.iterdir():
        if company_dir.is_dir():
            report_file = output_path / f"{company_dir.name}.html"
            if report_file.exists():
                companies_to_regenerate.append(company_dir.name)

    # 弹窗询问
    if companies_to_regenerate:
        result = messagebox.askyesno("缓存检查", message)
        if not result:
            use_cache = True
```

**Processor缓存处理:**
```python
# 检查缓存
if use_cache and report_file.exists():
    self._update_progress(f"使用缓存: {company_name}", idx, total)
    self.company_reports.append((company_name, report_file))
    continue
```
